# CLAUDE.md

Guidance for AI agents (and humans) working in this repo.

## What this is

whittle.tools — a collection of client-side developer utilities. React Router 7
(framework mode, SSR) on Cloudflare Workers, built with Vite. UI is Tailwind v4
+ shadcn/ui. Package manager is **pnpm**.

## Commands

- `pnpm dev` — dev server at http://localhost:5173
- `pnpm test` — Vitest unit tests
- `pnpm typecheck` — regenerates types (`wrangler types`, `react-router typegen`)
  then runs `tsc -b`. Run this after non-trivial changes.
- `pnpm build` — production build; `pnpm deploy` — deploy to Workers.
- `pnpm icons` — regenerate icons + OG image after changing the brand mark.

## Architecture

- **Tools are the domain model.** Each tool lives in `app/tools/<slug>/` and is
  registered in `app/tools/registry.ts`. Registration is the single source of
  truth — it drives the `/tools/:slug` route, home + directory listings, and the
  sitemap. To add a tool, follow the "Adding a tool" section in `README.md`.
- **Keep logic pure.** Put a tool's transformation in a dependency-free module
  (e.g. `splitter/transform.ts`) and unit test it. The `.tsx` component is just
  the UI around that function. Everything runs client-side; there is no backend.
- **Routing** is config-based in `app/routes.ts` (not file-based). Resource
  routes (`sitemap.ts`, `robots.ts`) export only a `loader` returning a Response.
- **SEO** goes through `app/lib/seo.ts` (`pageMeta`) plus per-route JSON-LD in
  the `meta` export. `app/lib/site.ts` holds branding/URL config.
- **Theming** is cookie-based and SSR-safe (`app/lib/theme.ts` +
  `components/theme-provider.tsx`). An inline script in `root.tsx` applies the
  theme before paint to avoid a flash. We do **not** use `next-themes`.
- **PWA**: `public/manifest.webmanifest` + `public/sw.js` (offline via
  `public/offline.html`), registered in production only by
  `components/register-service-worker.tsx`.

## Conventions

- Import alias: `@/*` → `app/*` (both `@/` and the template's `~/` resolve; new
  code uses `@/`).
- Add shadcn components with `pnpm dlx shadcn@latest add <name>`. They land in
  `app/components/ui/`. If a generated component imports `next-themes`, rewire it
  to `@/components/theme-provider` (see `ui/sonner.tsx`).
- Tabs for indentation (matches the template + Prettier config).
- Commits: Conventional Commits (`feat:`, `fix:`, `chore:`, `docs:`, `test:`).

## Gotchas

- **pnpm build approvals**: native deps (`esbuild`, `sharp`, `workerd`) are
  approved via `allowBuilds` in `pnpm-workspace.yaml`. Without approval,
  `pnpm install` exits non-zero and breaks `react-router build`.
- **`worker-configuration.d.ts`** is generated by `wrangler types` and committed.
  Cloudflare runtime types come from it, not `@cloudflare/workers-types`.
- After editing `wrangler.json` or adding bindings, run `pnpm cf-typegen`.
